1. Краткое описание

В этом задании реализован паттерн проектирования Visitor на примере системы “Умный дом”.
Смысл паттерна в том, чтобы можно было добавлять новые операции над объектами (устройствами), не изменяя сами классы устройств.

2. Что делает программа

Программа имитирует работу умного дома с тремя устройствами:
	•	лампа (Light)
	•	телевизор (TV)
	•	кондиционер (AirConditioner)

Для управления используются два визитора:
	•	TurnOnVisitor — включает все устройства
	•	TurnOffVisitor — выключает все устройства

3. Принципы “Clean Code”

При написании кода были использованы принципы из книги “Clean Code”:
	•	Чистые и понятные имена классов и методов
	•	Каждый класс отвечает только за одну задачу
	•	Код легко читать и расширять (можно добавить новые устройства без изменения старых классов)

4. Пример работы

После запуска программы выводится:
Smart Home System started!
Turning all devices ON
Light turned ON 
TV turned ON 
Air Conditioner turned ON 

Turning all devices OFF
Light turned OFF 
TV turned OFF 
Air Conditioner turned OFF 
All devices processed.

5. Вывод
Паттерн Visitor позволяет удобно добавлять новые действия с объектами без изменения их структуры.
Такой подход повышает гибкость кода и делает проект более чистым и поддерживаемым.
